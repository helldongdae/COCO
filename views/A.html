Problem A. 상상랩 자리 잡기

많은 한동인들은 모임 장소를로 잡을 때 상상랩을 이용한다. 어느 날 상상랩 앞을 지나가던 한동이는 사람들이 언제부터 언제까지 상상랩을 사용하기 원하는지 스케쥴을 쓴 종이가 붙어 있는 것을 보게 되었다. 그걸 본 한동이는 사람들의 스케줄이 주어 질 때, 중복 없이 최대 몇 팀이 하루에 상상랩을 쓸 수 있을지 궁금해졌다. 팀별로 스케줄이 주어 졌을 때, 하루에 최대 몇 팀이 상상랩을 이용 할 수 있는지 알아내는 프로그램을 작성하자.

입력

- 입력은 input.txt로 주어진다. 입력의 첫 줄에는 테스트 케이스의 개수가 주어진다.
- 각 테스트 케이스의 첫째 줄에는 상상랩을 예약하고 싶은 팀의 수인 정수 N이 주어진다.

0 <= N < 30

- 각 테스트 케이스의 둘째 줄부터 N+1줄까지 한 줄에 하나씩 모든 팀의 스케줄인 정수 M, K가 주어진다.
- 이는 해당 팀이 M시부터 시작해서 K시까지 모임을 가질 것이라는 의미이다.

0 <= M < K <= 24

- 모든 팀은 정확한 시간에 모임을 시작해서 정확한 시간에 모임을 마치고, 한 팀이 모임을 마친 직후, 다른 팀이 즉시 모임을 시작 할 수 있다.

출력

상상랩을 이용 가능한 최대 팀의 개수를 각 테스트 케이스 당 한 개의 정수로 한 줄에 하나씩 output.txt에 출력하여라.

Sample Input

1
5
0 10
1 3
2 5
6 10
12 20

Sample Output

3
